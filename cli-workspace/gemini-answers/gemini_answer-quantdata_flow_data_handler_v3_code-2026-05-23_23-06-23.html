Question

please read /Users/zhijiebian/Documents/Workplace/VSCodeProjects/MonkeyScripts/qd_flow_data_handler_v3.js thoroughly, and summize what it does with details

Answer

对该 Tampermonkey 脚本进行深度阅读后，其主要功能、工作流程和核心技术逻辑系统化整理如下：

1. 脚本定位与环境

  • 运行平台：QuantData Web 客户端（https://v3.quantdata.us/ 页面）。
  • 开发机制：基于 jQuery 进行 DOM 级轮询抓取，并利用 Tampermonkey 特权 API（如 GM_xmlhttpRequest 和本地 localStorage）实现前端页面与本地 API 服务的双向数据交换。

2. 核心功能模块与技术实现

A. DOM 数据抓取与转换 (processFlows)

脚本以固定时间间隔（实时模式下为 2秒，回填历史模式下为 4秒）轮询抓取 DOM 表格行 div.tr[class*="__tableRow"]：
* 字段提取：实时获取 Time（归一化为 HH:mm:ss 格式）、Contract（提取 Ticker, Expiration, Strike, Call/Put）、Sentiment（多空情绪）、Spot（现货价格）、Quantity、Price、Side（主动买盘 A/AA 或主动卖盘 B/BB）、Bid-Ask、Premium（以 $1,000 为单位换算）、Type（如 AUTO 算法单）、ConsType（合并类型）、Volume 和 OI（持仓量）。
* 开仓研判 (Open Action)：当 qty（成交量）大于 oi（昨日持仓量）时，脚本在数据中打上 action: 'Open' 标记，研判其为疑似主动开仓单。
* TOS 格式转换：计算并格式化生成 ThinkOrSwim 标准的合约编码（例如 .TSLA260612C425），便于下游系统精确匹配。

B. 数据去重与剪枝机制 (pruneTickersData)

  • 去重校验：新行抓取后，会比对本地缓存库中同 Ticker 数组内的字段是否已存在，防止因 DOM 延迟刷新产生重复数据。
  • 存储剪枝：为保证浏览器 localStorage 空间健康，脚本设置了 7天（10,080 分钟） 的历史去重窗口。
  • 每次追加新流水后，非大单的常规流水如果在内存中留存超过 60分钟 会自动从本地存储中剔除。
  • 达到“大单（Big Flow）”级别的数据将被保留 7 天，用来支持跨会话的大单总记与报表生成。

C. 多空资金流向统计 (addPremiumSummaryFromList)

脚本根据每笔订单的情绪（Sentiment）和 Call/Put 类型，自动归纳其资金属性（看多或看空）：
* 看多（Bullish）资金：Call 主动买入，或 Put 主动卖出。
* 看空（Bearish）资金：Put 主动买入，或 Call 主动卖出。
* 资金计算后汇总为 4 种累计资金量：
1. netPremiumsAll：总主动多空资金。
2. netPremiumsBig：大单主动资金。
3. netPremiumsBigBuy：大买单强多资金（侧重主动买盘 A/AA）。
4. netPremiumsNeutral：非主力通道（如 Mid 价成交）的被动/中性资金。

D. 同一时间多腿组合单合并 (findMultiLegsByTime)

如果同一秒内在同个 Ticker 上发生了多笔大单成交，脚本会自动将这些行组合为多腿组合单（Multi-Legs Spread）：
* 整体计算这组多腿订单的 Net Premium。
* 拼接该组合单的 TOS 编码链路，并在日志输出上额外标记 /Open（疑似开仓）和 /Neu（中性价成交）。

E. 重复小单追踪 (calcRepeatFlowSummaryData)

  • 主力在某些中小型行权价上可能不发送超大单，而是采用高频小单持续吸筹。
  • 脚本通过 repeatFlowSummary 持续记录同一 TOS 合约的累计订单数。
  • 当特定合约的累积金额达到 repeatFlowThreshod（默认 5,000 美金）且笔数 ≥ 3 时，自动标记为 Repeat Flow。

3. 外围系统联动与数据落库

  • 异常告警联动 (Port 5008)：
    当单笔 Net Premium 达到阈值，或高频重复单触发累积报警线时，通过 GM_xmlhttpRequest 将大单明细及推送报文发送至 http://127.0.0.1:5008/option_flow 控制器，从而触发外部 SMS 告警。
  • 数据自动归档入库 (Port 5005)：
    向本地 Python 数据库接口服务 http://127.0.0.1:5005/api/add-flows 发送 POST 请求将数据落库。
  • 13:30 (收盘后)：自动把今日筛选出来的全量大单明细 flows 发送入库。
  • 13:35：自动把今日各 Ticker 统计的 summaries 报表数据发送入库。

4. 界面前端可视化

  • 在 QuantData 页面顶部注入带有深色背景的控制面板。
  • 提供两个微型日志文本域：
  • bb_big_summary：大单组合与重复单监控日志。
  • bb_summary：各 Ticker（如优先监视的 TSLA、NVDA 等）的资金净流入大字报表。
  • 提供 Save Big Flows、Copy All Summary、Calc Current 等控制性按钮，方便日内分析及离线导出。
