Question

请详细记录现在day type的判断规则和强度计算规则 用@gem-doc

Answer

Day Type (日终定性) 判断规则

系统在盘尾依据盘前 (PM) 和常规交易时段 (RTH) 的 Delta 累积、价格变动以及吸收信号的统计，将盘面划分为五种基调（Day Type），默认为 Balanced（均衡震荡）。

1. Passive Buying (被动买方吸收)

判定满足以下任意一条即可：
- total_delta < -15,000 且 rth_price_change >= -10.0
*(全天大额净卖出，但 RTH 价格抗跌)*
- pm_delta < -5,000 且 pac >= 0.9 且 pm_price_change > 10.0 且 rth_price_change >= -10.0
*(盘前极强的主动抛压被全部吸收且价格逆势大涨，且 RTH 时段没有出现严重回调)*
- total_delta < -3,000 且 rth_price_change >= 10.0 且 dper < 0
*(全天净卖压且日内显著上涨，DPER为负验证了背离)*
- bottom_zone_delta < -500 且 bottom_zone_dper < 0
*(底部25%区间内，遭受卖盘抛压但价格不跌反涨，触发极限微观吸收)*
- top3_bear_30m_delta_sum < -5,000 且 top3_bear_30m_price_change_sum >= -10.0
*(空头倾泄火力最猛的 3 个 30分钟区间内，净卖单超过 5千手，但价格下跌不超过10点，中观吸收)*
- top10_bear_5m_delta_sum < -4,000 且 top10_bear_5m_price_change_sum >= -10.0
*(空头倾泄火力最猛的 10 个 5分钟区间内，净卖单超过 4千手，但价格下跌不超过10点，微观极值吸收)*

2. Passive Selling (被动卖方拦截)

判定满足以下任意一条即可：
- total_delta > 15,000 且 rth_price_change <= 10.0
- pm_delta > 5,000 且 pac >= 0.9 且 pm_price_change < -10.0 且 rth_price_change <= 10.0
- total_delta > 3,000 且 rth_price_change <= -10.0 且 dper < 0
*(全天净买盘且日内显著下跌，DPER为负验证了背离)*
- top_zone_delta > 500 且 top_zone_dper < 0
*(顶部25%区间内，遭受买盘拉升但价格不涨反跌，触发极限微观派发)*
- top3_bull_30m_delta_sum > 5,000 且 top3_bull_30m_price_change_sum <= 10.0
*(多头倾泄火力最猛的 3 个 30分钟区间内，净买单超过 5千手，但价格上涨不超过10点，中观吸收)*
- top10_bull_5m_delta_sum > 4,000 且 top10_bull_5m_price_change_sum <= 10.0
*(多头倾泄火力最猛的 10 个 5分钟区间内，净买单超过 4千手，但价格上涨不超过10点，微观极值吸收)*

3. Active Buying (主动买方推进)

  • total_delta > 15,000 且 rth_price_change >= 20.0 且 dper >= 1.5
    *(大额净买入，价格大幅上涨，且全天价格效率达到至少 1.5 点 / 1000手)*

4. Active Selling (主动卖方推进)

  • total_delta < -15,000 且 rth_price_change <= -20.0 且 dper >= 1.5
    *(大额净卖出，价格大幅下跌，且由于是向下推进，DPER 公式计算出的绝对效率需要大于等于 1.5)*

强度定量评分计算规则 (Scoring Breakdown)

系统基于 5 个核心维度对盘口强弱进行打分，总计 100 分。该评分用于量化当前 Day Type 趋势逻辑的可靠程度。

1. Zone DPER Score (40% 权重)

此项考核当前 Day Type 定性逻辑的专属关键微观区域内（如 Passive Buying 考量底部区间）的价格效率背离。
- Passive Buying: 必须满足 bottom_zone_delta < -500 且 bottom_zone_dper < 0，否则为 0。满足时按 min(|bottom_zone_dper| / 2.0, 1.0) * 40.0 计算。
- Passive Selling: 必须满足 top_zone_delta > 500 且 top_zone_dper < 0，否则为 0。满足时按 min(|top_zone_dper| / 2.0, 1.0) * 40.0 计算。
- Active / Selling: min(abs(dper) / 2.0, 1.0) * 40.0

2. Delta Score (10% 权重)

考核全天的净动能体量。
- 公式: min(abs(total_delta) / 30000.0, 1.0) * 10.0
- (全天绝对净 Delta 达到 30,000 手即满分 10 分)

3. Volume Score (10% 权重)

考核全天的市场参与度。
- 公式: min(total_volume / 2000000.0, 1.0) * 10.0
- (全天总成交量达到 200 万手即满分 10 分)

4. 30m Interval Absorb (20% 权重)

考核中观 30分钟 强抛压/买盘下的价格抗跌/抗涨程度（原 Divergence Score 改建）。
- Passive Buying: 必须满足 tb30d < -5000 且 tb30p >= -10.0，否则为 0。触发时：10.0 + min((|tb30d| - 5000) / 10000, 1) * 10.0
- Passive Selling: 必须满足 tu30d > 5000 且 tu30p <= 10.0，否则为 0。触发时：10.0 + min((|tu30d| - 5000) / 10000, 1) * 10.0
- Active 类: min(abs(rth_price_change) / 40.0, 1.0) * 20.0

5. 5m Interval Absorb (20% 权重)

考核微观极值 5分钟 的限价单拦截效果（原 Squeeze Score 改建）。
- Passive Buying: 必须满足 tb5d < -4000 且 tb5p >= -10.0，否则为 0。触发时：10.0 + min((|tb5d| - 4000) / 11000, 1) * 10.0
- Passive Selling: 必须满足 tu5d > 4000 且 tu5p <= 10.0，否则为 0。触发时：10.0 + min((|tu5d| - 4000) / 11000, 1) * 10.0
- Active / Balanced: 0分。

综合等级 (Strength Grade)

  • High: Total Score >= 75.0
  • Medium: Total Score >= 45.0
  • Low: Total Score < 45.0

Question

请加入对DPER的实例解释

Answer

DPER (Delta-Price Efficiency Ratio) 实例解析

DPER (价格效率比) 用于衡量“每一千手净 Delta 动能，能推升或打压多少点的价格”。它是判断市场中主动买卖（市价单）与被动拦截（限价冰山单）之间博弈强度的核心指标。

  • 基础公式: DPER = Price_Change / (Delta / 1000.0)

实例 1：正常的顺势推进 (Active Type)

假设在某个趋势向上的日子，RTH 时段内：
- 净 Delta = +15,000 手 (强力的市价买盘)
- 价格上涨 = +30 点
- 计算 DPER = 30 / (15000 / 1000) = +2.0
- 解析: 每一千手主动净买单，成功将价格推升了 2 点。由于 DPER >= 1.5，这表明多头推进效率极高，上方没有太大的被动限价单阻力，这是典型的 Active Buying (主动买方推进) 场景。

实例 2：底部的被动吸收 (Passive Buying 的常规防守)

假设在某个下跌放量的日子，价格打到了全天振幅底部的 25% 区间，此时我们观察这个底部微观区间 (Bottom Zone)：
- 底部区间 Delta = -10,000 手 (空头在底部仍在用市价单疯狂砸盘)
- 底部区间价格变动 = -5 点 (价格仅微弱下跌)
- 计算 Bottom Zone DPER = -5 / (-10000 / 1000) = +0.5
- 解析: 正常情况下 1 万手的巨大抛压可能会让价格跌去 15-20 点，但这里只跌了 5 点。说明底部有极厚的被动多头限价单在“托底”。空头的打压效率低下（DPER 仅为正数 0.5），这是多头吸筹的初步迹象，但价格依然在跟随 Delta 顺势下跌。

实例 3：底部的极端背离 (Passive Buying 满分场景)

同样是在底部 25% 区间，如果抛压被完全逆转：
- 底部区间 Delta = -10,000 手 (空头同样在疯狂砸盘)
- 底部区间价格变动 = +20 点 (价格不跌反涨！)
- 计算 Bottom Zone DPER = +20 / (-10000 / 1000) = -2.0
- 解析: 此时 DPER 为负数！说明不仅空头的巨额抛压没有让价格下跌，反而被下方的多头冰山单全数吃掉，并且多头的买气直接将价格逆势推高了 20 点。
- 打分应用: 在 Zone DPER Score 的 40分满分公式中：min(max(-bottom_zone_dper, 0.0) / 2.0, 1.0) * 40.0，带入 -2.0 后：max(-(-2.0), 0) = 2.0，2.0 / 2.0 = 1.0。最终得分 1.0 * 40.0 = 40.0。这代表着最完美的、具有极高反转胜率的底部被动吸筹形态，因此系统会给予 40 分满分的权重。

Question

强度定量评分计算规则 (Scoring Breakdown)

  1. Delta Score (20% 权重)

考核全天的净动能体量。
- 公式: min(abs(total_delta) / 30000.0, 1.0) * 20.0
- (全天绝对净 Delta 达到 30,000 手即满分)
3. Volume Score (20% 权重)

考核全天的市场参与度。
- 公式: min(total_volume / 2000000.0, 1.0) * 20.0
- (全天总成交量达到 200 万手即满分)

为什么全天的Delta和volume 需要的值越大 强度越大？背后的逻辑是什么？

Answer

在订单流 (Order Flow) 分析中，成交量 (Volume) 和 净动能 (Delta) 的绝对体量之所以与“日类型 (Day Type)”的强度（Strength Score）成正比，其核心逻辑可以归结为两个词：机构参与度 (Institutional Conviction) 与 结构的坚固性 (Structural Integrity)。

具体来说：

1. 为什么 Volume (总成交量) 越大，强度越高？

Volume 代表了全天真金白银的参与总规模和流动性深度。
* 低成交量环境 (Low Volume)：价格可能因为缺乏流动性（“真空地带”）而轻易被小资金推高或打压。这种行情通常是散户或短线程序化交易驱动的，产生的趋势和支撑/阻力非常脆弱，随时可能被反转。
* 高成交量环境 (High Volume)：当 ES 全天成交量逼近甚至超过 200 万手时，这意味着市场上最庞大的机构资金（Pension Funds, CTAs, 宏观对冲基金）正在大规模换手和建立仓位。在如此庞大的筹码交换后确立的 Day Type（无论是单边趋势还是底/顶部吸收），代表了机构级别的强烈共识。这种高成交量沉淀下来的支撑和阻力极其坚固，对后续几天的行情具有极强的指导意义。

2. 为什么 Absolute Delta (绝对净动能) 越大，强度越高？

Delta 衡量的是市价单买方和市价单卖方之间主动出击的失衡程度（Active Aggression）。全天 Delta 绝对值突破 30,000 手，代表市场存在极端的一边倒市价冲击。这种极端的失衡会极大增加最终 Day Type 结论的确定性：

  • 在顺势行情中 (Trend Day)：巨大的正 Delta 伴随大涨，或巨大负 Delta 伴随大跌，说明主动买盘/卖盘是毫无保留地倾泻，且没有任何力量能拦住他们。这是最纯粹的主动性单边碾压，趋势强度自然是极高的。
  • 在背离/吸收行情中 (Passive Buying / Passive Selling)：这是 Order Flow 最精髓的地方。如果全天 Delta 是巨大的负数（例如 -30,000 手，说明市价卖单在疯狂砸盘），但价格却没有跌甚至大涨（Passive Buying 吸收日），那么 Delta 的负值越大，说明被多头拦截的“有毒抛盘”就越多。多头竟然能用冰山限价单把多达 3万手的净砸盘全部生吃下来，这说明底部的机构护盘资金深不可测（深厚的 Liquidity Trap）。这种级别防守出来的支撑位，强度和含金量要远高于只吸收了 3000 手抛压的支撑位。

总结：
无论是顺势碾压还是逆势吸收，Volume 代表了战役的规模，Delta 代表了进攻方的火力。在一场超大规模、火力极猛的战役中，最终打出来的市场结构（Day Type）绝不可能是噪音，它必然是主力资金的真实意图所在，因此理应获得最高的强度评分。

passive_buying_rule_proposals

Question

这是现在Passive Buying的规则 结合我的所有的数据源 是否有其他合理的规则可以考虑

Answer

基于您的订单流（Order Flow）和拍卖市场理论（Auction Market Theory）的数据源（包括盘口深度失衡、机构大单、尾盘挤压和微观吸收频次等），我认为可以考虑引入以下几个非常合理且符合机构行为逻辑的 Passive Buying 补充规则：

1. 机构大单低位吸筹规则 (Institutional Block Trade Accumulation)

  • 规则条件：big_trade_score >= 1500 且 price_position_pct <= 45.0%
  • 逻辑解释：SingleTickBigTrade 记录的是大宗机构单。当机构在 RTH 偏低的价格区间（<= 45%）爆发出极高的大单权重（>= 1500），通常代表机构在低位以限价单（Iceberg / Limit Buy）大量承接抛压，以进行暗中吸筹。即使全天累计 Delta 因为散户恐慌性砸盘而呈现负数，机构大单的主导性也更强，可以直接定性为 Passive Buying。

2. 尾盘极速挤压/吸收规则 (EOD Absorption / Short Squeeze)

  • 规则条件：delta_15m < -4,000 且 price_change_15m >= 12.0
  • 逻辑解释：在常规交易时段（RTH）的最后 15 分钟里，如果市价单爆发出极大的净卖出量（<-4,000手），但价格不仅未跌，反而暴涨了 12.0 点 以上。这代表了典型的尾盘空头踩踏或极速吸收。主动卖方的市价抛单在收盘瞬间被巨量限价单全部拦截并顺势挤压，这是极强烈的多头信号，适合直接归为被动吸筹日。

3. 底部深度挂单托底规则 (Bottom Zone Depth Imbalance Support)

  • 规则条件：bottom_zone_delta < -1,000 且 bottom_zone_dper < 0 且 bottom_zone_avg_depth_imbalance > 0.15
  • 逻辑解释：除了考虑 DPER 量价背离，我们还可以结合 L2 挂单数据 (Depth Imbalance)。当价格处于全天底部 25% 时，即使有卖单砸盘，但只要：
    1. 底部确实有量价背离（dper < 0）。
    2. 买卖深度挂单比率（depth_imbalance）维持在 > 0.15（明显偏向买方）。
      这意味着在订单簿的更深处（Deep Book），机构在持续补充限价买单（Bids）以防守边界，从而验证了被动买盘的防御强度。

4. 底部高频冰山挂单规则 (Persistent Iceberg Signals)

  • 规则条件：bottom_zone_delta < -1,000 且 bottom_zone_dper < 0 且 micro_absorption_count >= 8
  • 逻辑解释：micro_absorption_count 记录了极值吸收频次。如果在量价已经背离的底部区域，高强度吸收信号触发频次非常多（>= 8），说明机构不是单次拦截，而是在该价格区间进行了持续、高频的冰山单（Iceberg Order）防御，这使得该底部的支撑效能更加可靠。
