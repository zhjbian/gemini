StudyPriceSpike 核心检测逻辑总结

该 Study 的核心目标是识别价格瞬间偏离正常波动范围的“异常成交”（Spike/Bad Prints）。以下是目前代码实现的技术路径：

1. 数据获取：深度 Tick 扫描 (Tick-Level Scan)

与普通指标不同，该 Study 不依赖 K 线（OHLC），而是使用显微镜级别的 instrument.getTicks(startTime, endTime) 获取整根 Bar 内部的所有原始成交记录。

⚠️ MW7 捕获失败的关键点：
如果 getTicks 返回 null 或空列表，循环将永远不会执行，即使 K 线上能看到影线，Study 也会因为拿不到底层数据而无法触发。

2. 基准设定：动态 ATR 边界 (ATR Bounds)

算法通过一个“基准价格”和 ATR 来定义什么是“正常波动”：

  • 基准价格 (closePrice)：通常使用特定锚点 Bar（如凌晨 1:02 AM）的最后一笔成交价。
  • 安全通道 (Bounds)：
    lowerBound = closePrice - 0.5 * ATR;
    upperBound = closePrice + 0.5 * ATR;

3. 判定逻辑：极值越界 (Out-of-Bounds Detection)

代码遍历每一个 Tick，检查其价格是否跌破或突破了安全通道：

if (tradePrice < lowerBound || tradePrice > upperBound) {
    // 检测到 Spike!
}

4. 信号去重与聚合 (Deduplication & Aggregation)

  • 唯一标识：使用 Ticker + Time + Price + Volume 生成唯一 Key，存入 spikesDetected 集合，防止同一个 Tick 被重复上报。
  • 聚合上报：将同一价格位的多个成交聚合为一个信号，并记录 maxAbsVolume（最大单笔单量）和 tradeCount（成交次数），避免信号轰炸。

5. 后端同步 (Backend Integration)

一旦确认 Spike，Study 会构造一个 JSON 对象并调用 sendNotificationToServer(spike)，通过 HTTP 异步发送至 Flask 后端，触发后续的统计和通知流程。
