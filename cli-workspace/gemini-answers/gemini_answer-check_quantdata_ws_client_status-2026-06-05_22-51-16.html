Answer

要检查 quantdata/ws_client.py 白天（RTH）是否正在后台运行，您有以下几种常用的方法：

1. 命令行检查进程 (最直接的方法)

您可以直接在 Mac 的 Terminal 中运行以下命令，如果有输出则说明进程正在运行（输出会显示进程 PID）：

pgrep -f "quantdata/ws_client.py"

或者使用 ps 命令查看更详细的运行状态和启动时间：

ps aux | grep "[q]uantdata/ws_client.py"

2. 检查日志文件是否有实时更新

在您的代码中，该进程的输出被重定向到了 bbt_data_web.log 中。您可以通过查看日志的最后几行，或者实时追踪日志输出来判断它是不是处于活跃状态：

# 查看最后20行日志
tail -n 20 /Users/zhijiebian/Documents/MyDoc/Finance/Current/Config/bbt_data_web.log

# 实时滚动监控日志（按 Ctrl+C 退出）
tail -f /Users/zhijiebian/Documents/MyDoc/Finance/Current/Config/bbt_data_web.log

3. 在 Python 代码中添加一个检查函数

如果您想在 daily_jobs.py 或其他 Python 脚本中写一段代码来自动检查它是否在运行，可以使用 os 或 subprocess：

import subprocess

def check_quantdata_ws_status():
    try:
        # pgrep 找到进程会返回 0，找不到会返回 1
        output = subprocess.check_output(["pgrep", "-f", "quantdata/ws_client.py"])
        print(f"✅ quantdata/ws_client.py 正在运行，PID: {output.decode().strip()}")
        return True
    except subprocess.CalledProcessError:
        print("❌ quantdata/ws_client.py 目前未运行！")
        return False

您可以根据是想手动检查（推荐方法1或2）还是代码监控（推荐方法3）来选择最适合的方案。
