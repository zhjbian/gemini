Question

How should the RTH_FH (First Hour) trading hour be integrated and backfilled, and how can the MotiveWave spike detection logic be stabilized to handle baseline drift and noise?

Answer

Institutional Signal Refinement and Spike Detection Stabilization

1. RTH_FH (First Hour) Integration & Backfill

The goal was to standardize the classification of trades occurring in the first hour of the market open (06:30 - 07:30 PT).

Technical Logic:

The logic was made inclusive of both boundaries to capture exactly one hour of trading.
* Ingestion Logic (db_query_order_flow.py):
python # Logic in order_flow_big_trade_add if trading_hour == 'RTH' and time(6, 30) <= trade_time <= time(7, 30): trading_hour = 'RTH_FH'
* Backfill Strategy: A 90-day backfill was executed using backfillRthFh which updated 21 records in the database to the RTH_FH classification.
* Backtest Engine (backtest_big_trades.py): Added a fallback to ensure historical RTH records in that window are treated as FirstHour during reporting.

2. MotiveWave Spike Detection Stabilization

The Java-based StudyPriceSpike.java was overhauled to eliminate "phantom" spikes and baseline corruption.

Baseline Resilience (Median Logic):

Replaced the 3-bar average with a Median of the last 11 bars. This ensures that even if a bar closes at a spike price, the baseline for subsequent bars remains anchored to the true market price.
* Intra-Day Guard: The median calculation strictly ignores previous days' data to prevent gap-related skewing.
java private double getMedianCloseIntraDay(DataSeries series, int index, int count) { String currentDateStr = BBU.getDateStrFrom(series.getStartTime(index)); List<Double> prices = new ArrayList<>(); for (int i = 1; i <= count && (index - i) >= 0; i++) { if (!BBU.getDateStrFrom(series.getStartTime(index - i)).equals(currentDateStr)) { break; // Stop at day boundary } prices.add((double) series.getClose(index - i)); } // ... sort and return median }

Noise Reduction (0.2% Hard Filter):

To eliminate low-volatility "junk" signals, a hard threshold was implemented.
* Threshold: Math.abs(tradePrice - closePrice) / closePrice < 0.002 (0.2%).
* Exception: SPY and QQQ are exempt from this filter to preserve sensitivity on index-level institutional prints.

3. Database Cleanup

A targeted cleanup of the spike_mw_agg table was performed to remove historical noise.
* Count: 796 records deleted.
* Query Criteria:
sql DELETE FROM spike_mw_agg WHERE ABS(price_change) <= 0.002 AND ticker NOT IN ('SPY', 'QQQ');

Impacted Code Artifacts:

  • Java: StudyPriceSpike.java (Median helper, hard filter, unified spotPrice).
  • Python: db_query_order_flow.py, backtest_big_trades.py.
  • Scratch: find_noise_spikes.py, delete_noise_spikes.py.
