Problem Analysis

After upgrading MotiveWave from v6 to v7, the mw_tick_parser_backfill_raw.py script was producing CSV tick data with severely incorrect BID/ASK side classification. The delta plotted by plot_es_volume_delta.py was completely wrong as a result.

Root Cause

MotiveWave v7 changed the .tick_data binary file format fundamentally:

Feature MW6 MW7
Byte Order Little-endian Big-endian
Header Variable length (~40 bytes) Fixed 28 bytes (magic=0x3281bb96)
Record size Variable (heuristic scan) Fixed 28 bytes
Side determination Direct byte[13] read (0=BID, 1=ASK) Compare trade_price against ask/bid prices
Record content side, time_delta, price, size time_delta, size, price, ask_price, ask_size, bid_price, bid_size

Evidence of the Bug

BID/ASK distribution comparison:

Data Source BID ASK Ratio
Old backup CSV (correct) 394,205 (50.4%) 387,179 (49.6%) Normal
Broken new CSV (MW6 parser on MW7 data) 944,369 (96.0%) 39,332 (4.0%) Severely wrong
Fixed new CSV (MW7 parser) 548,176 (55%) 448,789 (45%) Normal

MW7 Binary Format (Decoded)

Header (28 bytes, big-endian):

[0:4]   magic            (int32)  = 0x3281bb96
[4:8]   version          (int32)  = 1
[8:12]  record_count     (int32)
[12:20] start_timestamp  (int64)  milliseconds
[20:28] end_timestamp    (int64)  milliseconds

Record (28 bytes, big-endian):

[0:4]   time_delta_ms    (int32)  offset from hour start
[4:8]   trade_size       (int32)  number of contracts
[8:12]  trade_price      (float32) execution price
[12:16] ask_price        (float32) best ask at time of trade
[16:20] ask_size         (int32)  ask depth at best ask
[20:24] bid_price        (float32) best bid at time of trade
[24:28] bid_size         (int32)  bid depth at best bid

Side Determination Logic (MW7):
- If trade_price >= ask_price → ASK (buyer-initiated, lifting the ask)
- If trade_price <= bid_price → BID (seller-initiated, hitting the bid)
- Otherwise → midpoint trade, default to ASK

Fix Applied to mw_tick_parser_backfill_raw.py

Modified file: PyTools/order_flow_analysis/mw_tick_parser_backfill_raw.py

1. find_record_format() → detect_mw_version()

Replaced the old heuristic-only function with a version-aware detector that returns (header_size, rec_size, mw_version):

def detect_mw_version(data):
    """Detect MotiveWave binary format version.

    Returns: (header_size, rec_size, mw_version)
      - MW7: header=28, rec_size=28, version=7
      - MW6: header=variable, rec_size=variable, version=6
      - Unknown: (-1, -1, 0)
    """
    if len(data) < 56:
        return -1, -1, 0

    # --- Try MW7 format first ---
    magic = struct.unpack('>i', data[0:4])[0]
    if magic == 0x3281bb96:
        version_field = struct.unpack('>i', data[4:8])[0]
        record_count = struct.unpack('>i', data[8:12])[0]
        if version_field >= 1 and record_count > 0:
            rec = data[28:56]
            td = struct.unpack('>i', rec[0:4])[0]
            tsize = struct.unpack('>i', rec[4:8])[0]
            tprice = struct.unpack('>f', rec[8:12])[0]
            if 0 <= td < 3600000 and 10 < tprice < 100000 and 0 <= tsize < 1000000:
                return 28, 28, 7

    # --- Fall back to MW6 heuristic scan ---
    start = -1
    for i in range(40, 300):
        if i + 13 > len(data): continue
        td = struct.unpack('<i', data[i+1:i+5])[0]
        pr = struct.unpack('<f', data[i+5:i+9])[0]
        sz = struct.unpack('<i', data[i+9:i+13])[0]
        if 0 <= td < 3600000 and 10 < pr < 100000 and 0 <= sz < 1000000:
            start = i
            break
    if start == -1: return -1, -1, 0

    for rec_size in range(16, 40):
        next_i = start + rec_size
        if next_i + 13 > len(data): continue
        td = struct.unpack('<i', data[next_i+1:next_i+5])[0]
        pr = struct.unpack('<f', data[next_i+5:next_i+9])[0]
        sz = struct.unpack('<i', data[next_i+9:next_i+13])[0]
        if 0 <= td < 3600000 and 10 < pr < 100000 and 0 <= sz < 1000000:
            return start, rec_size, 6
    return start, -1, 0

2. get_first_tick_price() — Updated for MW7

rec_start, rec_size, mw_ver = detect_mw_version(data)
if rec_start != -1 and rec_size != -1:
    record = data[rec_start:rec_start+rec_size]
    if mw_ver == 7:
        price = struct.unpack('>f', record[8:12])[0]   # big-endian float at offset 8
    else:
        price = struct.unpack('<f', record[5:9])[0]     # little-endian float at offset 5
    return price

3. Main Parsing Loop — Dual format support

rec_start, rec_size, mw_ver = detect_mw_version(data)

if mw_ver == 7:
    record_count = struct.unpack('>i', data[8:12])[0]
    num_records = min(record_count, (len(data) - rec_start) // rec_size)
else:
    num_records = (len(data) - rec_start) // rec_size

for i in range(num_records):
    pos = rec_start + i * rec_size
    record = data[pos:pos+rec_size]

    if mw_ver == 7:
        # MW7: big-endian, 28-byte records
        time_delta = struct.unpack('>i', record[0:4])[0]
        t_time = hour_start_ms + time_delta
        volume = struct.unpack('>i', record[4:8])[0]
        price = struct.unpack('>f', record[8:12])[0]
        ask_price = struct.unpack('>f', record[12:16])[0]
        bid_price = struct.unpack('>f', record[20:24])[0]
        if price >= ask_price:
            side = "ASK"
        elif price <= bid_price:
            side = "BID"
        else:
            side = "ASK"  # midpoint trades default to ASK
    else:
        # MW6: little-endian
        time_delta = struct.unpack('<i', record[1:5])[0]
        t_time = hour_start_ms + time_delta
        price = struct.unpack('<f', record[5:9])[0]
        volume = struct.unpack('<i', record[9:13])[0]
        side = "ASK" if record[13] != 0 else "BID"
